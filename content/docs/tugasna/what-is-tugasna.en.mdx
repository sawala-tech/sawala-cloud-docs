**Tugasna** is Sawala Cloud's **work-tracking** product. You organize work into **boards** with Kanban columns, drop **items** onto those columns or park them in a project-wide **backlog**, and enrich each item with sub-items, labels, tags, custom fields, assignees, checklists, attachments, and threaded comments. It's a focused task board scoped to each project.

## When to choose Tugasna

Reach for Tugasna to track work alongside everything else in your project:

- A Kanban board for a team's tasks or a content pipeline.
- A backlog of ideas and to-dos you triage onto boards later.
- Lightweight project management that lives in the same place as your content, files, and forms.

## Core concepts

- **Board** — a container of columns for a project. New boards start with default statuses you can rename and reorder.
- **Status (column)** — a stage of work. Column order matters: an item in the last column counts as done.
- **Item** — a unit of work with a title, description, and details. Items can nest into **sub-items** to break work down.
- **Backlog** — a board-less pool. An item can sit in the backlog with no board or column until you **place** it onto one.
- **Labels vs tags** — both are coloured chips, but **labels belong to a board** (and clear when an item moves boards), while **tags belong to the project** (and follow an item everywhere). Use a tag for anything that should stick to the item across the whole project.

<Callout type="info">
Tugasna is managed entirely from the **dashboard** — there's no separate API key or SDK to set up. Just open your project and start a board.
</Callout>
